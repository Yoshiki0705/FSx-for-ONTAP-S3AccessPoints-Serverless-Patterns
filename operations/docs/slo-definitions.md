# SLO/SLI 定義 — Operations パターン

各 OPS パターン自体の信頼性目標を定義します。

---

## 共通 SLI

| SLI | 計測方法 | 目標 |
|-----|---------|------|
| レポート配信成功率 | Step Functions 成功 / 総実行 | ≥ 99.5% |
| レポート配信遅延 | Step Functions 完了時刻 - スケジュール時刻 | ≤ 15 分 |
| 推奨精度 (false positive 率) | 不要推奨数 / 総推奨数 | ≤ 5% |
| ONTAP API 到達率 | 200 応答 / 総リクエスト | ≥ 99% |
| Bedrock 推奨生成成功率 | 正常応答 / 総リクエスト | ≥ 95% |

---

## パターン別 SLO

### OPS1 (capacity-rightsizing)

| 目標 | SLO | エラーバジェット (月間) |
|------|-----|---------------------|
| 日次レポート配信 | 99.5% の日に 09:00 JST までに配信 | ~1.5 日/月 の遅延許容 |
| ティア推奨精度 | false positive < 5% | 20回推奨中 1回まで不要推奨可 |
| Level 2 承認→実行 | 承認後 10 分以内に実行完了 | — |

### OPS4 (snapshot-lifecycle)

| 目標 | SLO | 備考 |
|------|-----|------|
| 保持ポリシー違反検出 | 24 時間以内に検出 | 日次実行 |
| 削除推奨の正確性 | 法定期間内のスナップショットを削除推奨しない | 0 tolerance |
| Human Review 応答待ち | SLA 外 (人間依存) | タイムアウト 72h |

---

## エラーバジェット消費時のアクション

| 消費率 | アクション |
|:------:|----------|
| < 50% | 通常運用 |
| 50-80% | アラート閾値を緩和して推奨頻度を下げる |
| 80-100% | AutomationLevel を 1 段階下げる |
| > 100% | パターンを停止して根本原因分析 |
