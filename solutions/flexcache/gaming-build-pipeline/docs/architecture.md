# FC6: Gaming Build Pipeline — アーキテクチャ

## 概要

ゲーム開発ビルドパイプラインにおけるアセット品質チェック・ビルドログ分析の
自動化パイプライン。FlexCache によるグローバルスタジオ間のアセット共有を前提とする。

## 対応ゲームエンジン

| エンジン | アセット形式 | チェック項目 |
|---------|------------|------------|
| Unreal Engine 5 | .uasset, .umap | テクスチャ解像度、LOD 設定 |
| Unity | .prefab, .asset | メッシュ頂点数、マテリアル参照 |
| Godot | .tscn, .tres | シーン構造、リソース参照 |

## FlexCache の役割

- **グローバルアセット配信**: メインスタジオ → 各リージョンスタジオ
- **ビルドキャッシュ**: CI/CD パイプラインからの高速アセット読み取り
- **バージョン管理**: アセットバージョン間の差分配信

## Well-Architected 対応

| 柱 | 対応 |
|----|------|
| 運用上の優秀性 | ビルド品質メトリクス自動収集、ログ分析 |
| セキュリティ | アセットアクセス制御、ビルド成果物の暗号化 |
| 信頼性 | Step Functions Retry、品質ゲート |
| パフォーマンス効率 | FlexCache による WAN 転送削減、並列 QC |
| コスト最適化 | サーバーレス、FlexCache によるデータ転送コスト削減 |
