# FC4: Automotive CAE — アーキテクチャ

## 概要

自動車 CAE（Computer-Aided Engineering）シミュレーション結果の自動解析パイプライン。
LS-DYNA、STAR-CCM+、Nastran 等のソルバー出力を FSx ONTAP から S3 AP 経由で読み取り、
品質チェック・統計集計・レポート生成を自動化する。

## 対応ソルバー

| ソルバー | 出力形式 | 抽出メトリクス |
|---------|---------|--------------|
| LS-DYNA | d3plot, binout | エネルギー、変位、応力 |
| STAR-CCM+ | .sim, .csv | 流速、圧力、温度 |
| Nastran | .op2, .f06 | モード周波数、応力 |
| Abaqus | .odb | 変位、応力、ひずみ |
| OpenFOAM | postProcessing/ | 残差、力係数 |

## FlexCache の役割

- **設計→解析データ共有**: Origin（設計チーム）→ FlexCache（HPC クラスタ近傍）
- **解析結果の高速読み取り**: 大規模結果ファイル（数十 GB）のキャッシュ
- **マルチサイト共有**: 複数拠点の解析チーム間でデータ共有

## Well-Architected 対応

| 柱 | 対応 |
|----|------|
| 運用上の優秀性 | ソルバー出力の自動検出、品質メトリクス監視 |
| セキュリティ | 設計データのアクセス制御（NTFS ACL） |
| 信頼性 | Step Functions Retry、大規模ファイルのストリーミング処理 |
| パフォーマンス効率 | FlexCache による読み取り高速化、Range GET |
| コスト最適化 | サーバーレス処理、FlexCache によるWAN転送削減 |
