# パートナー向けワンページャー: Hybrid Cloud SnapMirror Sync

[日本語](partner-one-pager-ja.md) | [English](partner-one-pager.md)

> 1. このパターンは、ONTAP 上の既存エンタープライズファイルデータを、**別途 S3 データレイクを構築・複製することなく**、クラウド分析・BI で活用できることを示します。
> 2. SnapMirror は FSx for ONTAP を **near real-time のレプリケーテッドコピー** として最新状態に保ちます。
> 3. S3 Access Points がレプリケーションされたファイルデータを AWS サービスに公開し、**Amazon Quick / Quick Sight がビジネスユーザー向け UI** を提供します。

---

## ターゲット顧客

- オンプレミスで NetApp ONTAP をエンタープライズファイルデータに使用している組織
- フルデータ移行なしでクラウド分析・AI 機能を活用したい企業
- 既存の NAS 運用を維持しながら新しいワークロードを可能にしたい企業

## 顧客の課題

- NAS 上のエンタープライズファイルデータがクラウド分析・AI サービスから切り離されている
- S3 への完全コピーは二重管理のオーバーヘッドとデータの陳腐化を生む
- ビジネスユーザーが運用ファイルデータからセルフサービスでインサイトを得られない
- ソースファイルと分析システム間のデータ鮮度ギャップ

## ソリューション価値

| レイヤー | コンポーネント | 価値 |
|---------|--------------|------|
| レプリケーション | SnapMirror（スケジュール、near real-time） | 移行なしで AWS でのデータ可用性を継続的に確保 |
| ストレージ | FSx for NetApp ONTAP | NFS/SMB/S3 API 対応の AWS マネージド ONTAP |
| アクセス | S3 Access Points for FSx ONTAP | ファイルデータを S3 ネイティブの分析・AI サービスに接続 |
| 活用 | Amazon Quick / Quick Sight | ビジネスユーザーが自然言語やダッシュボードでデータを分析 |

## パートナーモーション

| フェーズ | アクティビティ | 成果物 |
|---------|--------------|--------|
| デモ | DEMO_MODE またはライブハイブリッド構成を実行 | 顧客がエンドツーエンドのデータフローを確認 |
| PoC | 顧客の ONTAP + FSx でデプロイ | レイテンシ、鮮度、アクセスパターンの検証 |
| 本番準備 | セキュリティレビュー、ガバナンス、モニタリング | アーキテクチャ決定記録 |
| マネージド運用 | 継続的な SnapMirror ヘルス管理、Quick 更新 | 運用ランブック |

## 3分間デモトークトラック

1. 「エンタープライズファイルデータはオンプレミスの ONTAP システム上にあります。」
2. 「SnapMirror が変更を AWS の FSx for ONTAP に継続的にレプリケーションします — 最短5分間隔のスケジュール、またはオンデマンドでトリガー可能です。」
3. 「S3 Access Points がそのファイルデータを S3 API 経由で公開します — 別バケットへのデータコピーは不要です。」
4. 「Amazon Quick / Quick Sight により、ビジネスユーザーがそのデータを即座に検索、可視化、活用できます。」
5. 「正本はオンプレミスに残ります。AWS は分析と AI のレイヤーを提供します。二重コピーなし、古いデータなし、移行不要です。」

## 機能の境界

- SnapMirror はスケジュールレプリケーション（最短5分間隔）であり、同期トランザクション共有ではない
- FSx S3 Access Points はファイルデータへの S3 API アクセスを提供 — 標準的な S3 バケットではない
- Amazon Quick は活用・インサイトレイヤー — 正本ではない
- 正本の定義は顧客のデータガバナンスに従う

## 成功指標

| 指標 | 説明 |
|------|------|
| Time to insight | ファイル作成から Quick で検索可能になるまでの時間 |
| データ鮮度 | SnapMirror レプリケーションラグ（スケジュール + オンデマンド） |
| 手動コピーの排除 | S3 同期ジョブや ETL パイプラインが不要 |
| セルフサービスユーザー数 | Quick ダッシュボードにアクセスするビジネスユーザー数 |
| 運用オーバーヘッド | データ準備ステップの削減 |
