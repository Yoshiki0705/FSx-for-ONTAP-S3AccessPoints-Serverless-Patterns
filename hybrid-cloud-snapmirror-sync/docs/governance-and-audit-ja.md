# ガバナンス、監査、データ分類

[日本語](governance-and-audit-ja.md) | [English](governance-and-audit.md)

## データ分類

| レイヤー | データの場所 | 分類 | 所有者 |
|---------|------------|------|--------|
| 正本（Source of Record） | オンプレミス ONTAP | 顧客が定義 | 顧客データオーナー |
| レプリケーション運用コピー | FSx for ONTAP (AWS) | コピー — 正本ではない | インフラチーム |
| 分析・可視化 | Amazon Quick データセット/ダッシュボード | 派生データ — 読み取り専用 | ビジネスアナリティクスチーム |

## 責任分担マトリクス

| 責任 | 担当 | 証跡 |
|------|------|------|
| ソースデータの整合性 | 顧客（オンプレミス） | ONTAP 監査ログ、ファイルパーミッション |
| SnapMirror レプリケーション正常性 | インフラチーム | SnapMirror 関係ステータス、転送ログ |
| FSx for ONTAP 管理 | クラウドインフラチーム | AWS CloudTrail、FSx 監査ログ |
| S3 Access Point ポリシー | セキュリティチーム | IAM ポリシー、S3 AP リソースポリシー |
| Amazon Quick アクセス制御 | アナリティクスチーム | Quick ユーザー/グループ管理 |
| データ分類ラベリング | データガバナンスチーム | メタデータタグ、命名規則 |

## アクセス制御レイヤー

```
[User] → [Amazon Quick IAM/SSO] → [S3 Access Point Policy] → [FSx File System Identity] → [ONTAP Volume]
```

各レイヤーが独立して認可を実施:
1. **Amazon Quick**: IAM Identity Center または Cognito によるユーザー認証
2. **S3 Access Point**: リソースベースポリシーでアクセス可能なプリンシパルを制限
3. **FSx file system identity**: S3 API コールを UNIX/NTFS ファイルパーミッションにマッピング
4. **ONTAP volume**: エクスポートポリシーと共有レベル ACL

## 監査証跡

| イベント | ログソース | 保持期間 |
|---------|-----------|---------|
| SnapMirror 転送開始/完了/失敗 | ONTAP 監査ログ、FSx CloudWatch | 設定可能 |
| S3 AP GetObject / ListObjects | AWS CloudTrail（データイベント） | デフォルト90日 |
| Quick ダッシュボードアクセス | Amazon Quick 監査ログ | Quick プランに準拠 |
| IAM 認証 | AWS CloudTrail | デフォルト90日 |
| ワンクリック同期トリガー | Sync Server audit.jsonl | アプリケーション管理 |

## 規制上の考慮事項

> **ガバナンス注意事項**: 本ドキュメントは技術的なガバナンスガイダンスとアーキテクチャパターンを提供するものです。法的、コンプライアンス、または規制上のアドバイスを構成するものではありません。拘束力のあるコンプライアンス判断については、資格のある専門家にご相談ください。

規制産業（医療、金融、公共セクター）向け:
- データレジデンシー要件の確認（レプリケーションが指定リージョン内に留まること）
- 暗号化要件の検証（保存時 KMS、転送時 TLS）
- 監査保持期間が規制上の最低要件を満たすことを確認
- レプリケーションデータの分類がデスティネーション環境のセキュリティポスチャを超えないことを確認

## AI / BI 利用の境界

このパターンはレプリケーションデータに対する AI および BI 探索をサポートできます。
ただし、レプリケーションデータは以下の条件を満たさない限り、高影響の自動意思決定に使用してはなりません:

- データ分類がユースケースに対してレビュー・承認済み
- 鮮度 SLO が現在満たされている（`slo-design.md` 参照）
- AI/BI 出力に正本タイムスタンプが表示されている
- 高影響アクションに対するヒューマンレビューワークフローが定義済み
- AI 生成の推奨事項に対する監査証跡が有効

> このパターンは、レプリケーションデータが自動的に AI-ready であることを前提としていません。
> BI、AI、または運用分析にデータが準備できているかを判断するための
> 鮮度、ガバナンス、オブザーバビリティシグナルを提供します。
