# ビジネスユーザーガイド

[日本語](business-user-guide-ja.md) | [English](business-user-guide.md)

このガイドは、FSx for ONTAP からレプリケーションされたデータを基に Amazon Quick / QuickSight で構築されたダッシュボードやインサイトにアクセスするビジネスユーザー向けです。

## 意思決定の前に確認すべきこと:

1. **正本タイムスタンプ** — 元データはいつ作成・更新されたか？
2. **最終レプリケーションタイムスタンプ** — データはいつ AWS にコピーされたか？
3. **ダッシュボード更新タイムスタンプ** — Quick / QuickSight が最後にデータを取得したのはいつか？
4. **データ鮮度ステータス** — データは Fresh / Stale / Unknown のいずれか？
5. **ヒューマンレビューフラグ** — このインサイトはアクション前にヒューマンレビューが必要か？
6. **既知の制限事項** — このダッシュボードから結論付けられないことは何か？

## ダッシュボードラベル（必須）

すべてのダッシュボードまたは Quick の回答には以下を表示すること:

| ラベル | 例 |
|--------|---|
| データソース | FSx for ONTAP 上のレプリケーションコピー |
| 正本タイムスタンプ | 2026-06-10 10:00:00 JST |
| 最終レプリケーション日時 | 2026-06-10 10:05:12 JST |
| ダッシュボード更新日時 | 2026-06-10 10:06:00 JST |
| データ経過時間 | 6分 |
| 鮮度ステータス | ✅ Fresh（SLO 内） |
| 正本か？ | ❌ いいえ — これはレプリケーションコピーです |

## このデータとは

- エンタープライズファイルデータの near real-time レプリケーションビュー
- トレンド分析、パターン検出、運用状況把握に適している
- スケジュール SnapMirror レプリケーション + オンデマンドトリガーで更新

## このデータではないもの

- 正本（正本はオンプレミス ONTAP）
- ゼロレイテンシのリアルタイムデータ（レプリケーションは最短5分スケジュール）
- 規制・コンプライアンス決定のための検証済みデータ（追加レビューなしでは）
- ヒューマン確認なしの即時自動アクションに適したデータ

## 安全な利用ガイドライン

1. 具体的な値を引用する前に、レプリケーションタイムスタンプを必ず確認する
2. 精度が重要な場合はソースシステムと照合する
3. 業務、財務、安全に影響するアクションにはヒューマンレビューを要求する
4. 古い、欠落している、または不整合に見えるデータがあれば報告する
5. ダッシュボードの値をソース検証なしに契約上の証拠として使用しない
